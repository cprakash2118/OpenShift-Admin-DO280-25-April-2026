Unit-3-# Authentication & Authorization
       - Configure Identity Providers (IdPs).
	   
	   - What is Identity Provider (IdP)?
	     - While login to OCP cluster (Token based) IdP verify access (Authenication access).
           $ oc login -u user -p password APU-URL 
           Example:-
           $ oc login -u admin -p redhatocp https://api.ocp4.example.com:6443  
		   
	   - IdP alway define in OAuth service (OAuth is an built-in function w.r.t OCP).
	   - OAuth is an identity Broker 
	     $ oc get oauth cluster 
		 $ oc get oauth cluster -o yaml
		 
		 FYI
		 - Whenever login to OCP cluster (Token based), user always be part of some namespace (project)
		 - By default it's "default" namespace.
		 
		user1-5/redhat -> secretName=secret-name-new-team  IdP Name= IdP-htpasswd-type-NEW-Team
		
		- Secret for IdP must be on "openshift-config" namespace only.
