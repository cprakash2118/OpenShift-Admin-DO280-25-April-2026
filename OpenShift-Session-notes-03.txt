Prod Setup for OCP cluster:-
- 3 master nodes - Run cluster service/application w.r.t k8s/ocp.
- N worker nodes [ worker nodes later convert to storage/infra/custom etc] - End user applications.
- Login to OCP cluster.
  - OAuth service used for "Authentication & Authorization" - Identity Broker.
    FYI- OAuthv2.0 - On behalf of OpenShift it communicate with other IdP (Identity Providers).
	
  - Authentication methods:-
    1-) TOken Based auth [OAuth Based authentication]
	    FYI
		- Authentication always done by IdP, the IdP exchange the token with OAuth.
		- OAuth involve with RBAC - OAuth provides the authorization.
		
    2-) Certificate Based auth  (Backdoor & auditor level - It bypass OAuth) 	
	
	
  - Identity Providers:-
    - htpasswd (Local IdP) 
	- LDAP & LDAP Group Sync 
	- AD - Active Directory 
	- OpenIDConnect (OIDC) - AWS IAM, RHSSO, PingIdentity etc

What is OCP cluster:-
Prod -> master + worker nodes [3 master nodes & N worker nodes]
LAB  -> Master + worker nodes [1 node OCP cluster - master node also act as worker - all-in-one-node]
        Dual role - master01 -> [ master, worker ] 

Deploy applications/operators - Belongs to system (services).
  - one pod/container -> belongs to master01
  - If 3 node cluster (3 master nodes).
  - Three pods/containers -> belongs to master01-03 
  
    OAuth (service) -> Running 
	oauth-pod01 -> master01 
	oauth-pod02 -> master02 
	oauth-pod03 -> master03 
	
	FYI- 
	- Each services w.r.t OpenShift also associated with respective operator.
	- Operator (w.r.t k8s/ocp operator) is also an application which invoke k8s/ocp API to manage k8s/ocp resources like (Oauth PODS,Deployment,etc).
	- All applications/services belongs to specific "namespace".
	
	$ oc get pods -n openshift-authentication
	  Outcome -> Display all OAuth service running here w.r.t namespace="openshift-authentication"
	
    $ oc get pods -n openshift-authentication-operator 

- Where to do the changes w.r.t Authentication (htpasswd, LDAP, AD, OIDC etc).
  $ oc get oauth /cluster 
  $ oc get oauth /cluster -o yaml 
  $ oc get oauth /cluster -o yaml > my-oauth.yaml
  $ vim my-oauth.yaml 
  $ oc apply/replace -f my-oauth.yaml   (This type of changes called "declarative method").
  $ oc get pods -n openshift-authentication 
    Note: pods should restart if any changes.
  
  
- This changes read by oauth-operator running w.r.t openshift-authentication-operator namespace.
- oauth-operator restart the oauth-pod01/02/03 in rolling update.
